## Project Overview

The Procure-to-Pay cycle is an end-to-end procurement process that begins with identifying material demand and ends with successful vendor payment. This project uses a fictional manufacturing company, **TechNova Electronics Pvt Ltd**, to demonstrate real-world SAP implementation.

The solution integrates purchasing, inventory movement, invoice verification, payment processing, and analytics dashboards.

---

## SAP Modules Used

- SAP MM (Materials Management)  
- SAP FI (Financial Accounting)  
- SAP Business Data Cloud  
- SAP Analytics Cloud  

---

## Complete P2P Process Flow

1. Material Requirement Planning  
2. Purchase Requisition  
3. Vendor Selection / RFQ  
4. Purchase Order Creation  
5. Goods Receipt  
6. Invoice Verification  
7. Vendor Payment  

---

## SAP Transaction Codes Used

| Transaction Code | Description |
|------------------|-------------|
| ME51N | Create Purchase Requisition |
| ME21N | Create Purchase Order |
| ME41 / ME49 | RFQ & Vendor Comparison |
| MIGO | Goods Receipt |
| MIRO | Invoice Verification |
| F110 | Automatic Payment Run |

---

## Business Scenario

TechNova Electronics Pvt Ltd requires procurement of Lithium Battery Modules for production. SAP helps manage the complete cycle:

- Requirement generated through planning  
- Purchase requisition created  
- Vendor selected  
- Purchase order issued  
- Goods received into inventory  
- Invoice matched with PO and GR  
- Vendor payment released on time
